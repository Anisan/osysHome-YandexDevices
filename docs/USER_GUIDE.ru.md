# YandexDevices - Руководство пользователя

![YandexDevices Icon](../static/YandexDevices.png "Плагин YandexDevices")

## Назначение

`YandexDevices` интегрирует osysHome с Yandex Quasar IoT и Яндекс Станциями.

После настройки модуль позволяет:

- авторизовать доступ к аккаунту Yandex через QR;
- импортировать и обновлять список станций и IoT-устройств;
- получать значения capabilities и свойств устройств;
- связывать значения устройств со свойствами и методами объектов osysHome;
- отправлять команды из osysHome обратно в устройства Yandex;
- воспроизводить cloud TTS на выбранных станциях.

> [!IMPORTANT]
> Интеграция двусторонняя: Yandex -> osysHome (обновления значений) и osysHome -> Yandex (управление устройствами).

---

## Обзор интерфейса

Основная страница модуля:

```text
/admin/YandexDevices
```

Верхние действия:

1. `Update` - обновляет станции и устройства из Yandex cloud.
2. `Authorization` - статус и запуск авторизации по QR.
3. `Settings` - параметры опроса и обновлений.

Вкладки:

- `Stations` - настройки станций и TTS.
- `Devices` - устройства и привязки capabilities.

---

## Чек-лист быстрого запуска

- [ ] Откройте `/admin/YandexDevices`.
- [ ] Перейдите в `Authorization` и завершите вход по QR.
- [ ] Нажмите `Update`, чтобы загрузить станции и устройства.
- [ ] Во вкладке `Stations` настройте TTS на нужных станциях.
- [ ] Во вкладке `Devices` откройте устройство и задайте привязки.
- [ ] В `Settings` включите опрос, если нужны периодические обновления.

---

## Авторизация (QR-сценарий)

Откройте:

```text
YandexDevices?op=auth
```

Далее:

1. Нажмите `QR code`.
2. Сканируйте код приложением Yandex.
3. Нажмите `Continue` для подтверждения.
4. Убедитесь, что статус стал `Authorized`.

Если cookie/токен устарели, нажмите `Reset` и повторите вход.

> [!WARNING]
> Без успешной авторизации запросы к API Yandex не будут работать.

## Вкладка Stations

В списке станций отображаются:

- название и иконка;
- минимальный уровень `Min level say`;
- состояние `Online`/`Offline`;
- время `Updated`.

### Форма редактирования станции

Поля:

| Поле | Назначение |
| --- | --- |
| `Title` | Название станции в базе модуля |
| `Platform` | Идентификатор платформы Yandex |
| `IOT id` | IoT-идентификатор |
| `IP` | Опциональный IP станции |
| `Token` | Device token для локального управления |
| `TTS` | `No`, `Local (not work)`, `Cloud` |
| `Min level SAY` | Минимальный уровень для `say()` |

Если токена нет, используйте `Generate token` в карточке станции.

## Вкладка Devices

В списке устройств доступны:

- название и иконка;
- тип Yandex-устройства;
- комната;
- IoT ID;
- время последнего обновления.

В карточке устройства можно настроить:

- `Update period` (индивидуальный период опроса);
- привязки по каждой capability/property;
- флаг `Readonly` для направления связи.

---

## Как работают привязки capability

Примеры ключей:

- `devices.capabilities.on_off`
- `devices.capabilities.range.temperature`
- `devices.properties.float.temperature`
- `devices.properties.event.open`

Каждая запись может быть привязана к:

- `linked_object` + `linked_property`
- `linked_object` + `linked_method`

### Привязка к свойству

- значение из Yandex записывается в свойство объекта osysHome;
- если связь не read-only, обратные изменения свойства могут отправляться в устройство.

### Привязка к методу

При изменении значения вызывается метод с параметрами:

```json
{
  "NEW_VALUE": "...",
  "OLD_VALUE": "...",
  "DEVICE_STATE": 1,
  "UPDATED": "timestamp",
  "MODULE": "YandexDevices"
}
```

## SAY и TTS

Действие:

```text
say(message, level=0, args=None)
```

Фильтры перед отправкой:

- на станции должен быть включен `tts`;
- должен быть задан `min_level`;
- `level` сообщения должен быть >= `min_level`.

Режимы:

- `tts = 1` (`Local`) - в текущем UI помечено как неработающее.
- `tts = 2` (`Cloud`) - выполняется через сценарий Yandex.

Длинные сообщения в cloud-режиме разбиваются на предложения и отправляются по частям.

## Настройки

Глобальные параметры в модальном окне:

| Настройка | Значение |
| --- | --- |
| `Enable get device data` | Включает периодический опрос в цикле модуля |
| `Default update period device data (seconds)` | Период по умолчанию для устройств без собственного интервала |
| `Update only linked devices` | Опрос только устройств, у которых есть хотя бы одна привязка |

- количество станций;
- количество устройств.

## Диагностика

### Статус остается `Not authorized`

Проверьте:

- QR-поток завершен до шага `Continue`;
- cookie не были сброшены;
- сервер имеет доступ к endpoint-ам Yandex.

### Устройства есть, но значения не обновляются

Проверьте:

- включен `Enable get device data`;
- корректен `Update period` у устройства;
- авторизация аккаунта еще действительна.

### Привязки не срабатывают

Проверьте:

- указаны существующие object/property или object/method;
- после сохранения привязок устройство действительно опрашивалось;
- у capability реально изменяется значение.

### Обратное управление не работает

Проверьте:

- capability не отмечена как `Readonly`;
- тип capability поддерживает исходящие команды в текущей реализации.
